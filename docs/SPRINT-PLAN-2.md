## 📦 Phase 1: Sprint 1 遗留项清零

### 1.1 技能学习消耗金/木
**当前问题**：Sprint 1 计划里写了"技能学习消耗金钱木材"，但实际只需要技能点，没有花金/木

**修改内容**：
- [ ] HeroTowerLogic.learnSkill() 加金/木消耗检查
- [ ] 消耗表：1级免费，2级 100金，3级 300金+1木，4级 800金+3木，5级 2000金+5木
- [ ] UI 中技能按钮显示消耗（灰显不够的）
- [ ] 需要从 Game.ts 传入 EconomyManager 引用或回调

**验收**：技能学习时扣除金/木，不够时按钮灰显+提示

### 1.2 英雄选择界面显示人口成本
**当前问题**：选英雄时看不到占多少人口，选了 hard 英雄占 4 人口但不知道

**修改内容**：
- [ ] 英雄卡片上显示人口成本（如"👥 占2人口" / "👥 占4人口"）
- [ ] easy 英雄标绿色推荐标签

**验收**：选英雄前能看到人口代价

### 1.3 Tab 键实时 DPS 排行
**当前问题**：Sprint 1 Phase 3.2 计划了"游戏中按 Tab 查看实时统计"，没有实现

**修改内容**：
- [ ] Tab 键打开/关闭 DPS 统计面板
- [ ] 显示每座塔的击杀数排行（前 5 名）
- [ ] 显示英雄输出占比
- [ ] 显示总场上怪物 / 当前波 / 游戏时长

**验收**：Tab 键弹出半透明统计面板

### 1.4 环境粒子
**当前问题**：Sprint 1 Phase 2.2 计划了"环境粒子（萤火虫/花瓣）"，没有实现

**修改内容**：
- [ ] 场景中加 15-20 个缓慢飘动的绿色光点（萤火虫效果）
- [ ] 用 PointsMaterial 或 Sprite，不影响性能

**验收**：场景中有微弱的飘浮光点

### 1.5 光环塔浮空动画
**当前问题**：Sprint 1 Phase 2.3 计划了"光环塔浮空效果加上下浮动"，没有实现

**修改内容**：
- [ ] 光环塔(aura_attack/aura_speed)的顶部八面体加上下浮动
- [ ] 光环圈加缓慢旋转

**验收**：光环塔有明显的浮空动画，一眼能和普通塔区分

## 📦 Phase 2: 性能 + 架构

### 2.1 TopBar DOM 缓存
**当前问题**：updateTopBar() 每帧调用 querySelector × 9，浪费性能

**修改内容**：
- [ ] createPlayingUI 时缓存所有 DOM 引用（gold/wood/wave/pop/score/enemy/speed/nextWave/bossTimer）
- [ ] updateTopBar 直接用缓存引用

**验收**：updateTopBar 中无 querySelector 调用

### 2.2 Game.ts 拆分
**当前问题**：Game.ts 1084 行，包含游戏逻辑+UI+输入+渲染，难以维护

**修改内容**：
- [ ] 提取 `UIManager.ts` — 所有 HTML UI 创建/更新（菜单/商店/顶栏/面板/结算）
- [ ] 提取 `InputManager.ts` — 键盘/鼠标事件处理
- [ ] Game.ts 只保留核心游戏循环和系统协调（目标 <400 行）

**验收**：Game.ts < 400 行，3 个新文件各自职责清晰，编译通过+功能不变

## 📦 Phase 3: 数值验证 + 修正

### 3.1 E2E 通关测试
**当前问题**：经济系统大改后，不确定普通难度能否通关

**修改内容**：
- [ ] 用加速(x4)模式试玩到 20 波，检查：
