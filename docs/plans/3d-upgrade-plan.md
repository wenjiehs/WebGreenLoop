# 绿色循环圈 3D 化改造方案

## 🎯 目标

将当前 2D 几何形状风格升级为**类魔兽争霸3的低多边形 3D 风格**，同时**不改动游戏逻辑层**（经济/波次/伤害/克制/技能等）。

## 🏗️ 技术方案：双层架构

```
┌──────────────────────────────────────────┐
│         Three.js 3D 渲染层 (新增)         │
│  地形/塔模型/怪物模型/弹道/粒子/光照       │
├──────────────────────────────────────────┤
│         Bridge 桥接层 (新增)              │
│  将 Phaser 游戏逻辑的坐标/状态同步到 3D    │
├──────────────────────────────────────────┤
│      Phaser 2D 游戏逻辑层 (保持不变)      │
│  塔/怪物/弹道/波次/经济/技能/英雄/光环     │
└──────────────────────────────────────────┘
```

**核心思路：Phaser 继续跑游戏逻辑，Three.js 只负责渲染。**

- Phaser 的 Canvas/WebGL 渲染隐藏或缩小为缩略图
- Three.js 作为主视觉层，使用斜 45° 俯视视角（类魔兽3）
- Bridge 层每帧从 Phaser 读取所有实体的 (x, y, state)，映射到 Three.js 的 3D 世界

### 任务 1：项目基建（预计 2h）

**目标**：引入 Three.js，搭建双层架构骨架

### 任务 2：3D 地形系统（预计 3h）

**目标**：用 Three.js 渲染闭合跑道地形，替代 Phaser 的 drawMap

| # | 子任务 | 说明 |
|---|---|---|
| 2.1 | 草地平面 | 带噪声纹理的绿色平面，程序化生成（Perlin noise 颜色变化） |
| 2.2 | 路径网格 | 根据 PathManager.getPathTiles() 生成石砖路面模型（微凸+贴图） |
| 2.3 | 路径边缘 | 路面和草地之间的过渡斜坡/泥土带 |
| 2.4 | 建造网格线 | 可建造区域半透明绿色网格线 |
| 2.5 | 环境装饰 | 随机树/灌木/石头/花（低多边形程序化生成或 GLTF 模型） |
| 2.6 | 光照 | 太阳方向光 + 环境光 + 柔和阴影 |

**坐标映射**：
- Phaser: (px, py) 像素坐标，原点左上角
- Three.js: (x, 0, z) 世界坐标，y轴朝上
- 映射公式: `threeX = (phaserX - GAME_WIDTH/2) * SCALE`, `threeZ = (phaserY - GAME_HEIGHT/2) * SCALE`

### 任务 8：Bridge 同步层（预计 3h）

**这是最关键的一层，连接 Phaser 逻辑和 Three.js 渲染**

| # | 子任务 | 说明 |
|---|---|---|
| 8.1 | EntitySync 基类 | 监听 Phaser 实体的 position/state，同步到 Three.js 对象 |
| 8.2 | TowerSync | Tower 创建时 → 3D 模型生成；升级 → 模型更新；销毁 → 模型移除 |
| 8.3 | EnemySync | Enemy 每帧同步位置/朝向/血量/状态（减速/中毒/隐形） |
| 8.4 | ProjectileSync | Projectile 弹道同步（2D直线 → 3D抛物线） |
| 8.5 | HeroTowerSync | 英雄塔的技能施放视觉、Buff 发光等 |
| 8.6 | EventSync | 监听游戏事件（波次开始/Boss出场/胜利/失败）触发3D动画 |
| 8.7 | 建造预览 | 鼠标悬停时在 3D 世界中显示半透明塔预览 |

**同步流程（每帧）**：
```
Phaser update() → 游戏逻辑更新 → GameBridge.sync() → 遍历所有实体
  → 塔: position/level/selected/auraActive → Three.js TowerModel
  → 怪: position/hp/slow/poison/invisible → Three.js EnemyModel  
  → 弹: position → Three.js Projectile
  → UI: 保持 HTML overlay 不变
```

## 🚀 推荐执行顺序

```
Phase A (基础能跑): 任务1 → 任务8 → 任务2
  → 3D 地形渲染出来，怪物/塔还是2D小点，但已经在3D世界中同步移动

Phase B (有模型): 任务3 → 任务4
  → 塔和怪物替换为3D模型，视觉上已经像魔兽3了

Phase C (效果打磨): 任务5 → 任务6 → 任务7
  → 弹道/闪电/爆炸3D化 + 摄像机交互 + 光影后处理
```

**Phase A 做完就已经有"3D 感"了（大约 8h），后续可以逐步打磨。**

## ⚠️ 风险和备选

| 风险 | 应对 |
|---|---|
| 性能：大量怪物(100+)+塔+弹道 | 使用 InstancedMesh 批量渲染同类模型 |
| 模型制作耗时 | Phase A 先用简单几何体，后续替换 GLTF |
| 移动端兼容 | Three.js 默认支持 WebGL2，降级到 WebGL1 |
| Phaser+Three.js 同时跑 | Phaser 可设置 `renderType: Phaser.HEADLESS` 不渲染 |

## 🎨 魔兽3 视觉参考

| 元素 | 魔兽3风格特征 |
|---|---|
| 色调 | 饱和度中等，偏暖色，非写实 |
| 地形 | 大色块+简单纹理，草地有深浅变化 |
| 建筑 | 粗犷线条，明显的木/石结构，顶部旗帜/火把 |
| 单位 | 夸张比例（大头小身），动作幅度大 |
| 光照 | 暖色太阳光，柔和阴影，夜间蓝色月光 |
| 粒子 | 简单但量大，法术效果明亮 |
